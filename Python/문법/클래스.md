## 클래스

클래스(class)는 [객체 지향 프로그래밍](https://github.com/976520/Study/tree/main/Object%20Oriented%20Programming)의 핵심 개념으로, 데이터(속성)와 그 데이터를 처리하는 함수(method)를 하나로 묶은 틀이다. 클래스에서 실제로 생성된 개별 실체를 instance(instance) 혹은 객체(object)라고 한다.

```python
class Dog:
    pass

콩 = Dog()
print(type(콩)) #출력: <class '__main__.Dog'>
```

파이썬에서 클래스명은 PascalCase(각 단어의 첫 글자를 대문자로)로 작성하는 것이 관례이다.

1.  생성자

    생성자(constructor)는 instance가 생성될 때 자동으로 호출되는 특수 method이다. 파이썬에서는 `__init__` method가 생성자 역할을 하며, instance의 초기 상태를 설정하는 데 쓰인다.

    ```python
    class Dog:
        def __init__(self, name, age):
            self.name = name
            self.age = age

    bori = Dog("보리", 3)
    print(bori.name) #출력: 콩
    print(bori.age)  #출력: 3
    ```

    `self`는 생성된 instance 자기 자신을 가리키는 참조이다. 모든 instance method의 첫 번째 매개변수로 반드시 명시해야 하며, method를 호출할 때 파이썬이 자동으로 전달하므로 직접 인수를 넣을 필요는 없다.

2.  instance 변수와 클래스 변수

    1.  instance 변수

        `self.변수명`의 형태로 선언되며, 각 instance마다 독립적으로 존재하는 변수이다. 한 instance의 instance 변수를 수정해도 다른 instance에는 영향을 주지 않는다.

        ```python
        class Dog:
            def __init__(self, name):
                self.name = name

        dog1 = Dog("콩")
        dog2 = Dog("보리")
        print(dog1.name) #출력: 콩
        print(dog2.name) #출력: 보리
        ```

    2.  클래스 변수

        클래스 내부, method 바깥에 선언되며, 해당 클래스로 생성된 모든 instance가 공유하는 변수이다. `클래스명.변수명` 또는 `self.변수명`으로 접근할 수 있지만, 명확성을 위해 클래스명을 통한 접근을 권장한다.

        ```python
        class Dog:
            species = "Canis lupus familiaris"

            def __init__(self, name):
                self.name = name

        dog1 = Dog("콩")
        dog2 = Dog("보리")
        print(dog1.species) #출력: Canis lupus familiaris
        print(dog2.species) #출력: Canis lupus familiaris

        Dog.species = "개"
        print(dog1.species) #출력: 개
        print(dog2.species) #출력: 개
        ```

        클래스 변수는 instance 생성 횟수를 추적하거나, 모든 instance가 공유해야 하는 상수를 저장할 때 유용하다.

3.  method

    method(method)는 클래스 내부에 정의된 함수이다. 역할에 따라 세 종류로 나뉜다.

    1.  instance method

        가장 일반적인 method로, 첫 번째 매개변수로 `self`를 받아 instance 변수에 접근하거나 수정할 수 있다.

        ```python
        class Dog:
            def __init__(self, name, age):
                self.name = name
                self.age = age

            def bark(self):
                print(f"{self.name}이 짖습니다: 왈왈!")

            def get_info(self):
                return f"이름: {self.name}, 나이: {self.age}"

        my_dog = Dog("콩", 3)
        my_dog.bark()             #출력: 콩이 짖습니다: 왈왈!
        print(my_dog.get_info())  #출력: 이름: 콩, 나이: 3
        ```

    2.  클래스 method

        `@classmethod` 데코레이터를 사용하며, 첫 번째 매개변수로 instance 대신 클래스 자체를 나타내는 `cls`를 받는다. 클래스 변수에 접근하거나, instance를 다양한 방식으로 생성하는 팩토리 method를 만들 때 주로 사용된다.

        ```python
        class Dog:
            count = 0

            def __init__(self, name):
                self.name = name
                Dog.count += 1

            @classmethod
            def get_count(cls):
                return f"현재까지 생성된 개의 수: {cls.count}"

        dog1 = Dog("콩")
        dog2 = Dog("보리")
        print(Dog.get_count()) #출력: 현재까지 생성된 개의 수: 2
        ```

    3.  정적 method

        `@staticmethod` 데코레이터를 사용하며, `self`나 `cls`를 받지 않는다. 클래스나 instance의 상태에 접근하지 않지만, 클래스와 논리적으로 연관된 기능을 묶어두고 싶을 때 사용한다.

        ```python
        class MathUtils:
            @staticmethod
            def add(a, b):
                return a + b

            @staticmethod
            def is_even(n):
                return n % 2 == 0

        print(MathUtils.add(3, 4))    #출력: 7
        print(MathUtils.is_even(10))  #출력: True
        ```

4.  캡슐화

    캡슐화(encapsulation)는 객체의 내부 데이터를 외부로부터 보호하는 것이다. 파이썬은 네이밍 컨벤션(naming convention)으로 접근 수준을 표현한다.

    1.  public

        아무런 접두사가 없는 일반 변수 및 method로, 클래스 외부에서 자유롭게 접근할 수 있다.

        ```python
        class Person:
            def __init__(self, name):
                self.name = name # public

        p = Person("은지")
        print(p.name) #출력: 은지
        ```

    2.  protected

        단일 언더바(`_`)를 접두사로 붙인 변수 및 method이다. 외부에서 직접 접근하는 것을 지양해야 한다는 약속이지만, 문법적으로 강제되지는 않는다. 주로 서브클래스(하위 클래스)에서 접근하도록 의도된 경우에 사용한다.

        ```python
        class Person:
            def __init__(self, name, age):
                self._age = age

        p = Person("은지", 25)
        print(p._age) #출력: 25 (접근은 가능하지만 권장하지 않음)
        ```

    3.  private

        이중 언더바(`__`)를 접두사로 붙인 변수 및 method이다. 파이썬은 이름 맹글링(name mangling)을 통해 `_클래스명__변수명`으로 이름을 변경하여 외부에서의 직접 접근을 어렵게 만든다.

        ```python
        class BankAccount:
            def __init__(self, balance):
                self.__balance = balance

            def get_balance(self):
                return self.__balance

        account = BankAccount(100000)
        print(account.get_balance()) #출력: 100000
        # print(account.__balance)   # AttributeError 발생
        print(account._BankAccount__balance) #출력: 100000 (맹글링된 이름으로는 접근 가능)
        ```

5.  property

    `@property` 데코레이터를 이용하면 method를 마치 변수처럼 접근하게 만들 수 있다. private 변수에 대한 getter, setter, deleter를 정의하여 유효성 검사나 부가 로직을 포함하면서도 깔끔한 인터페이스를 유지할 수 있다.

    ```python
    class Person:
        def __init__(self, age):
            self.__age = age

        @property
        def age(self):
            return self.__age

        @age.setter
        def age(self, value):
            if value < 0:
                raise ValueError("나이는 0 이상이어야 합니다.")
            self.__age = value

        @age.deleter
        def age(self):
            del self.__age

    p = Person(25)
    print(p.age) #출력: 25

    p.age = 30
    print(p.age) #출력: 30

    # p.age = -1 # ValueError: 나이는 0 이상이어야 합니다.
    ```

    setter가 정의되지 않은 경우 해당 프로퍼티는 읽기 전용이 된다.

6.  상속

    상속(inheritance)은 기존 클래스(부모 클래스, 슈퍼클래스)의 속성과 method를 물려받아 새로운 클래스(자식 클래스, 서브클래스)를 만드는 기능이다. 코드 재사용성을 높이고, 클래스 간의 계층 관계를 표현하는 데에 쓰인다.

    1.  단일 상속

        클래스 선언 시 괄호 안에 부모 클래스를 명시한다.

        ```python
        class Animal:
            def __init__(self, name):
                self.name = name

            def breathe(self):
                print(f"{self.name}이가 숨을 쉽니다.")

        class Dog(Animal):
            def bark(self):
                print(f"{self.name}이가 짖습니다: 왈왈!")

        my_dog = Dog("콩")
        my_dog.breathe() #출력: 콩이가 숨을 쉽니다.
        my_dog.bark()    #출력: 콩이가 짖습니다: 왈왈!
        ```

    2.  `super()`

        `super()`는 부모 클래스를 참조하는 객체를 반환한다. 자식 클래스에서 부모 클래스의 method를 호출할 때 사용하며, 특히 자식 클래스의 `__init__`에서 부모의 초기화 코드를 실행할 때 자주 쓰인다.

        ```python
        class Animal:
            def __init__(self, name):
                self.name = name

        class Dog(Animal):
            def __init__(self, name, breed):
                super().__init__(name)
                self.breed = breed

        my_dog = Dog("콩", "비숑 프리제")
        print(my_dog.name)  #출력: 콩
        print(my_dog.breed) #출력: 비숑 프리제
        ```

    3.  method overriding

        자식 클래스에서 부모 클래스의 method를 같은 이름으로 재정의하는 것을 method overriding이라고 한다. 호출 시 자식 클래스의 method가 우선된다.

        ```python
        class Animal:
            def speak(self):
                print("...")

        class Dog(Animal):
            def speak(self):
                print("왈왈!")

        class Cat(Animal):
            def speak(self):
                print("야옹!")

        animals = [Dog(), Cat(), Animal()]
        for animal in animals:
            animal.speak()

        '''
        출력:
            왈왈!
            야옹!
            ...
        '''
        ```

    4.  다중 상속

        파이썬은 여러 부모 클래스로부터 동시에 상속받는 다중 상속을 지원한다. 괄호 안에 부모 클래스를 쉼표로 구분하여 나열한다.

        ```python
        class Flyable:
            def fly(self):
                print("날아갑니다.")

        class Swimmable:
            def swim(self):
                print("헤엄칩니다.")

        class Duck(Flyable, Swimmable):
            def quack(self):
                print("꽥꽥!")

        꾸왁스 = Duck()
        꾸왁스.fly()   #출력: 날아갑니다.
        꾸왁스.swim()  #출력: 헤엄칩니다.
        꾸왁스.quack() #출력: 꽥꽥!
        ```

        두 부모 클래스에 동일한 이름의 method가 있을 경우, 파이썬은 MRO(Method Resolution Order, method 결정 순서)에 따라 왼쪽에 명시된 부모 클래스를 우선한다. `클래스명.mro()`로 순서를 확인할 수 있다.

        ```python
        print(Duck.mro())
        #출력: [<class 'Duck'>, <class 'Flyable'>, <class 'Swimmable'>, <class 'object'>]
        ```

    5.  `isinstance()`와 `issubclass()`

        `isinstance(obj, cls)`는 obj가 cls의 instance인지(또는 cls의 서브클래스의 instance인지)를 확인한다. `issubclass(sub, parent)`는 sub가 parent의 서브클래스인지를 확인한다.

        ```python
        class Animal: pass
        class Dog(Animal): pass

        my_dog = Dog()
        print(isinstance(my_dog, Dog))    #출력: True
        print(isinstance(my_dog, Animal)) #출력: True (상속 관계 포함)
        print(issubclass(Dog, Animal))    #출력: True
        ```

7.  특수 method

    특수 method(special method)는 이름 앞뒤에 이중 언더바(`__`)가 붙어 있으며, 던더 method(dunder method) 또는 매직 method(magic method)라고도 불린다. 파이썬의 내장 연산자나 함수와 연동되어 동작한다.

    1.  `__str__`과 `__repr__`

        `__str__`은 `print()` 등 사람이 읽기 좋은 문자열 표현을 위해 호출되고, `__repr__`은 객체를 재현할 수 있는 공식적인 문자열 표현을 위해 호출된다.

        ```python
        class Point:
            def __init__(self, x, y):
                self.x = x
                self.y = y

            def __str__(self):
                return f"Point({self.x}, {self.y})"

            def __repr__(self):
                return f"Point(x={self.x}, y={self.y})"

        p = Point(3, 4)
        print(p)      #출력: Point(3, 4)
        print(repr(p)) #출력: Point(x=3, y=4)
        ```

    2.  비교 연산자 method

        비교 연산자에 대응하는 특수 method를 정의하면, 해당 연산자를 instance에 사용할 수 있다.

        | method | 연산자 |
        |---|---|
        | `__eq__(self, other)` | `==` |
        | `__ne__(self, other)` | `!=` |
        | `__lt__(self, other)` | `<` |
        | `__le__(self, other)` | `<=` |
        | `__gt__(self, other)` | `>` |
        | `__ge__(self, other)` | `>=` |

        ```python
        class Point:
            def __init__(self, x, y):
                self.x = x
                self.y = y

            def __eq__(self, other):
                return self.x == other.x and self.y == other.y

            def __lt__(self, other):
                return (self.x**2 + self.y**2) < (other.x**2 + other.y**2)

        p1 = Point(1, 2)
        p2 = Point(1, 2)
        p3 = Point(3, 4)
        print(p1 == p2) #출력: True
        print(p1 < p3)  #출력: True
        ```

    3.  산술 연산자 method

        산술 연산자에 대응하는 특수 method를 정의하여 instance 간의 연산을 가능하게 할 수 있다.

        ```python
        class Vector:
            def __init__(self, x, y):
                self.x = x
                self.y = y

            def __add__(self, other): # + 연산자
                return Vector(self.x + other.x, self.y + other.y)

            def __mul__(self, scalar): # * 연산자
                return Vector(self.x * scalar, self.y * scalar)

            def __str__(self):
                return f"Vector({self.x}, {self.y})"

        v1 = Vector(1, 2)
        v2 = Vector(3, 4)
        print(v1 + v2)  #출력: Vector(4, 6)
        print(v1 * 3)   #출력: Vector(3, 6)
        ```

    4.  컨테이너 관련 method

        `__len__`, `__getitem__`, `__setitem__`, `__contains__` 등을 정의하면 instance를 시퀀스나 매핑처럼 다룰 수 있다.

        ```python
        class NumberList:
            def __init__(self, numbers):
                self.numbers = numbers

            def __len__(self):
                return len(self.numbers)

            def __getitem__(self, index):
                return self.numbers[index]

            def __contains__(self, item):
                return item in self.numbers

        nl = NumberList([10, 20, 30, 40])
        print(len(nl))     #출력: 4
        print(nl[2])       #출력: 30
        print(20 in nl)    #출력: True
        ```

    5.  `__call__`

        `__call__`을 정의하면 instance를 함수처럼 호출할 수 있다.

        ```python
        class Multiplier:
            def __init__(self, factor):
                self.factor = factor

            def __call__(self, value):
                return value * self.factor

        double = Multiplier(2)
        print(double(5))  #출력: 10
        print(double(9))  #출력: 18
        ```

8.  추상 클래스

    추상 클래스(abstract class)는 직접 instance를 생성할 수 없으며, 상속받는 서브클래스가 반드시 구현해야 할 method를 강제하는 클래스이다. `abc` 모듈의 `ABC`와 `@abstractmethod`를 사용한다.

    ```python
    from abc import ABC, abstractmethod

    class Shape(ABC):
        @abstractmethod
        def area(self):
            pass

        @abstractmethod
        def perimeter(self):
            pass

    class Circle(Shape):
        def __init__(self, radius):
            self.radius = radius

        def area(self):
            return 3.14159 * self.radius ** 2

        def perimeter(self):
            return 2 * 3.14159 * self.radius

    # shape = Shape()  # TypeError: 추상 클래스는 instance 생성 불가
    c = Circle(5)
    print(c.area())      #출력: 78.53975
    print(c.perimeter()) #출력: 31.4159
    ```

    `@abstractmethod`가 붙은 method를 서브클래스에서 구현하지 않으면 해당 서브클래스도 추상 클래스가 되어 instance를 생성할 수 없다.

9.  데이터 클래스

    `@dataclass` 데코레이터를 사용하면 `__init__`, `__repr__`, `__eq__` 등의 특수 method를 자동으로 생성해준다. 주로 데이터를 저장하는 목적의 클래스를 간결하게 작성할 때 유용하다.

    ```python
    from dataclasses import dataclass

    @dataclass
    class Point:
        x: float
        y: float

    p1 = Point(1.0, 2.0)
    p2 = Point(1.0, 2.0)
    print(p1)        #출력: Point(x=1.0, y=2.0)
    print(p1 == p2)  #출력: True
    ```

    필드에 기본값을 설정하거나, `frozen=True`로 불변 객체를 만들 수도 있다.

    ```python
    from dataclasses import dataclass, field

    @dataclass(frozen=True)
    class Config:
        host: str = "localhost"
        port: int = 8080
        tags: list = field(default_factory=list)

    config = Config()
    print(config.host) #출력: localhost
    print(config.port) #출력: 8080
    # config.host = "example.com" # FrozenInstanceError: 수정 불가
    ```
